# Afferent

Cost-of-service / revenue intelligence app.

## Stack

- **Vite 8** + **React 19** + **TypeScript 5**
- **TanStack Router** (file-based; route tree generated by `@tanstack/router-plugin/vite`)
- **Zustand** for the build-model store (`useBuildState`)
- **Hono** server for AI-parse endpoints, using the Anthropic SDK
- `@fontsource(-variable)` for Inter Tight + IBM Plex Sans/Mono
- CSS variable design tokens (no CSS-in-JS lib, no Tailwind)

Path alias `@/*` resolves to the repo root (see `tsconfig.json`). Import as
`@/components/...`, `@/features/...`, `@/lib/...`.

## Running it

```bash
npm install
npm run dev
```

`npm run dev` runs two processes in parallel via `concurrently`:

- **client** — Vite on `http://localhost:3000`, with `/api/*` proxied to the API
- **api**    — Hono on `http://localhost:8787`

Set `ANTHROPIC_API_KEY` in `.env.local` to enable the `/api/ai/parse-*` endpoints
(PDF imports degrade gracefully without it).

Other scripts: `npm run build` (Vite production build), `npm run preview`
(serve the build), `npm run typecheck` (`tsc --noEmit`).

```
src/
  main.tsx                 # mounts <RouterProvider/>
  index.css                # global styles
  routes/                  # file-based TanStack Router routes
    __root.tsx             # root layout: <TopBar/> + <Outlet/>
    index.tsx              # /
    gap.tsx                # /gap
    monitoring.tsx         # /monitoring
    build.tsx, build/*     # /build, /build/{services,salary,operating,cap,
                           #   workload,costs,policy,feestudy,benchmark}
    annual.tsx, annual/*   # /annual, /annual/{refresh,changes}
    export/fee-study.tsx   # /export/fee-study
  pages/                   # page bodies, imported by the matching route file
  routeTree.gen.ts         # generated by router-plugin (gitignored)

server/                    # Hono API on :8787 (proxied as /api from Vite)
  index.ts                 # Hono entry + route registration
  aiParseRunner.ts         # shared Anthropic PDF parse helper
  aiParse{Services,Salary,Operating,Cap,Workload,Fees}.ts

components/
  ui/                      # Btn, Stat, RecoveryMeter, DeptChip, Icon,
                           # SectionLabel, NodeEyebrow, Formula, ExportMenu, …
  layout/                  # Page, TopBar, SubNav, PageHeader
  table/                   # DataTable (sortable, filterable, drilldown-ready)

features/                  # screen-level components, one folder per feature
  home/                    # EntryCard, AuditTrail
  revenue-gap/             # AnswerHeader, DriverBreakdown, DeptRecoveryChart, TopFixesTable
  build/                   # per-screen tables and panels for the build workflow
  annual/                  # annual-update screens
  imports/                 # PageImportDrawer (Claude PDF + paste JSON)
  _shared/                 # cross-screen helpers (StatusRow, …)

lib/
  store.ts                 # Zustand build store + useBuildState() hook
  calc.ts                  # pure derivations (FBHR, fee comparisons, policy impact)
  types.ts                 # domain types
  format.ts                # fmt.dollars / dollarsK / pct / int
  signals.ts               # signalFor() — recovery % → tone
  data/                    # seed data, one file per domain noun
  ai/                      # client-side adapters for /api/ai/parse-* endpoints
  parse/                   # shared ExtractionResult / SourceLineage types
  export/                  # XLSX + PDF fee-study export helpers

styles/

```

## Design tokens

All visual constants live in `styles/tokens.css` as CSS custom properties.
Never hardcode a hex, never invent a new font size — pick from the scale. The
token ladder, top to bottom:

- **Paper** — `--canvas`, `--paper`, `--paper-2`, `--paper-3` (page bg sits *below* card bg).
- **Ink** — `--ink`, `--ink-2`, `--ink-3`, `--ink-4`.
- **Accent / brand** — `--accent`, `--accent-tint`, `--navy`, `--charcoal`.
- **Signal** — `--neg`, `--warn`, `--pos` (plus matching `-tint` variants).
- **Type scale** — `--t-l1` through `--t-l9` with paired weight tokens.
- **Spacing** — `--s-1` (4px) through `--s-8` (40px).
- **Geometry** — `--page-max`, `--page-padX`, `--section-gap`.

Numeric content uses `.num` (tabular nums) or `.mono` (full mono).
Display headings use `.display` (Inter Tight + tight tracking).
