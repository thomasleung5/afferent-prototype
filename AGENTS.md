# Architecture

This is a Vite + React + TanStack Router SPA, with a small Hono server for the AI extract endpoint. There is no Next.js — do not introduce `next/link`, `next/navigation`, `app/` directory routing, server components, or `"use client"` directives.

- `src/main.tsx` mounts `<RouterProvider/>`; build-model state is read through the persisted Zustand store in `lib/store.ts`.
- File-based routes live in `src/routes/`; page bodies live in `src/pages/`.
- The route tree is generated to `src/routeTree.gen.ts` by `@tanstack/router-plugin/vite` (gitignored).
- The path alias `@/*` resolves to the project root.
- `npm run dev` launches Vite on :3000 (proxied `/api → :8787`) and Hono on :8787 in parallel.

# Styling

- Plain CSS only. Use the existing CSS variables / design tokens defined in `src/index.css` (and `styles/tokens.css`).
- Do **not** introduce Tailwind, CSS-in-JS, `styled-components`, `emotion`, or any new styling system.
- Follow the existing patterns in `components/`, `features/`, and `src/pages/` — inline style objects driven by tokens are the norm for one-off layouts; recurring UI lives in `components/ui/` and `components/layout/`.

# Generated files

- `src/routeTree.gen.ts` is generated by the TanStack Router Vite plugin. Do not edit by hand — running `npm run build` (or `npm run dev`) regenerates it. It is gitignored.

# Verification

- TypeScript: `npm run typecheck`.
- Unit tests are fixture-style TypeScript files under `lib/__tests__/`, `server/__tests__/`, and `lib/data/__tests__/`, run with `tsx` through `npm run test:*` scripts.
- Prefer the nearest targeted `npm run test:<name>` for focused changes; use `npm test` for the full chain.
- Browser smoke tests (Playwright chromium): `npm run test:smoke`. First-time setup: `npm run test:smoke:install`.

# Local environment

- Local API dev usually needs `.env.local` with `NODE_ENV=development` and `AUTH_DEV_BYPASS=1` so the server's Supabase JWT gate relaxes for development.
- AI parsing requires `ANTHROPIC_API_KEY`.
- Production requires the Supabase env vars (`SUPABASE_URL`, `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`) and `ALLOWED_ORIGINS`.
